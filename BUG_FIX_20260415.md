# 🔧 Bug 修复报告 - 2026-04-15

## ❌ 发现的问题

### 问题 1: NameError - Path 未定义
**错误信息**: 
```
NameError: name 'Path' is not defined
```

**错误位置**: `mod_manager_view.py` 第 574 行
```python
tooltip = f"Local Mod\nFolder: {Path(mod.folderPath).name}\n{mod.name}"
```

**根本原因**: 
- Python 使用了缓存的 .pyc 文件
- 虽然源代码已添加 `from pathlib import Path`，但旧缓存未更新

---

### 问题 3: 无法添加 mod 到 Active Mods

- UI 组件未能正确构建

## ✅ 解决方案

### 步骤 1: 清除 Python 缓存（重要！）

```powershell
cd B:\Work\python\goh-mod-manager-cn

# 删除所有 __pycache__ 目录
Get-ChildItem -Path . -Directory -Filter "__pycache__" -Recurse | Remove-Item -Recurse -Force

# 删除所有 .pyc 文件
```

## ⚠️ 常见问题

### Q: 为什么修改了代码还是报错？
A: Python 会缓存 .pyc 文件，必须删除 `__pycache__` 目录才能生效。

### Q: 如何确认修复成功？
A: 查看调试日志中的 `Is workshop:` 输出：
- 本地 mod 应该显示 `Is workshop: False`
- Steam mod 应该显示 `Is workshop: True`

### Q: Active Mods 还是不工作？
A: 确保：
1. 已清除缓存
2. 程序完全关闭后重新启动
3. 查看是否有其他错误日志
